# Contributing to This Project

Thank you for your interest in contributing!

This repository is developed by Group 10 as part of UBC DSCI 310 and follows a structured GitHub workflow to ensure collaboration, reproducibility, and high-quality contributions.

---

## Code of Conduct

All contributors must follow our [Code of Conduct](CODE_OF_CONDUCT.md).

We expect respectful communication and inclusive behavior at all times.

---

## Reporting Issues

If you find a bug or would like to request a feature:

1. Open an issue in the GitHub issue tracker.
2. Clearly describe:
   - The problem or feature request
   - Steps to reproduce (if applicable)
   - Expected vs actual behavior

Please use GitHub Issues instead of private messages so that discussions remain documented.

---

## Making Contributions (Pull Requests)

We follow a GitHub Flow workflow.

### Step 1: Create a Branch

Create a new branch from `main`:

- `docs/<description>` for documentation
- `feat/<description>` for new features
- `fix/<description>` for bug fixes

---

### Step 2: Make Your Changes

- Keep changes focused and small.
- Write meaningful commit messages.
- Ensure code runs inside the Docker container.

---

### Step 3: Open a Pull Request

1. Push your branch.
2. Open a Pull Request into `main`.
3. Describe what you changed and why.

---

### Step 4: Code Review

- At least one teammate must review your PR.
- Address feedback before merging.
- Do not merge without review.

---

## Development Environment

This project uses Docker to ensure reproducibility.

Please follow instructions in the README to build and run the container.
